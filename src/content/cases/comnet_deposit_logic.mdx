## Описание

Comnet - интернет-провайдер, чьи пользователи регулярно пополняют счёт через личный кабинет. Форма оплаты - это один из самых частых сценариев использования продукта, поэтому любое неудобство здесь напрямую влияет на опыт каждого абонента.

</div>

<div class="lang-en">

## Description

Comnet is an internet service provider whose users regularly top up their accounts through their personal accounts. The payment form is one of the most common ways the product is used, so any inconvenience here directly affects each subscriber’s experience.

</div>

<img src="/images/cases/comnet/6.webp" alt="uzcard and visa ui design" />

<div class="lang-ru">

## Проблема

Старая форма была минималистичной до неудобства: поле для суммы и набор иконок платёжных систем. Никаких сохранённых карт, никакой истории — каждый раз нужно было заново вводить данные карты вручную. При этом разные типы карт (Visa, Uzcard, Humo) никак не различались интерфейсом, что создавало путаницу.

</div>

<div class="lang-en">

## Problem

The old form was so minimalist it was practically unusable: just a field for the amount and a set of payment system icons. There were no saved cards and no transaction history—you had to manually re-enter your card details every single time. On top of that, the interface didn’t distinguish between different card types (Visa, Uzcard, Humo), which caused confusion. 

</div>

<img src="/images/cases/comnet/7.webp" alt="change card for pay" />
<img src="/images/cases/comnet/8.webp" alt="select card for pay" />

<div class="lang-ru">

## Решение и результат

Я переработал форму с нуля. Теперь пользователь видит свои сохранённые карты и выбирает нужную в один клик — интерфейс сам определяет тип карты и показывает только нужные поля. Для Uzcard и Humo запрашивается привязанный номер телефона, для Visa — CVV и имя держателя. Добавлено управление картами: можно сделать карту основной, включить автоплатёж или удалить её прямо из выпадающего меню. Дизайн соответствует брендбуку Comnet. Форма стала самостоятельным инструментом, а не просто полем для ввода. Пользователи больше не вводят данные карты при каждом пополнении. Поддержка локальных платёжных систем (Uzcard, Humo) сделала интерфейс привычным для узбекского рынка. Возможность подключить автоплатёж снизила количество случаев, когда абонент забывал пополнить счёт.

</div>

<div class="lang-en">

## Solution and result

I redesigned the form from scratch. Now users can view their saved cards and select the one they need with a single click—the interface automatically detects the card type and displays only the relevant fields. For Uzcard and Humo, the linked phone number is requested; for Visa, the CVV and cardholder’s name are required. Card management has been added: you can set a card as the primary one, enable auto-payment, or delete it directly from the dropdown menu. The design aligns with Comnet’s brand book. The form has become a standalone tool, not just an input field. Users no longer have to enter their card details every time they top up. Support for local payment systems (Uzcard, Humo) has made the interface familiar to the Uzbek market. The ability to enable auto-pay has reduced the number of instances where subscribers forget to top up their accounts. 

</div>
